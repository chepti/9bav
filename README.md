# גוש קטיף — פרויקט תיעוד קהילתי

אתר לתיעוד הגירוש מגוש קטיף ומצפון השומרון.
תושבים שגורשו מעלים מדיה וסיפורים על הבית שלהם, ומודרטורים מנהלים ומאשרים.

בהשראת פרויקט "7.10 360" של כאן, בשפה עיצובית של תקווה ושיקום (כתום/צהוב/ענבר) ולא של אבל.

## הרצה מקומית

```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # בנייה לתיקיית dist/
npm run preview  # תצוגה מקדימה של הבנייה
```

## מבנה

```

    store.js       שכבת נתונים ריאקטיבית (localStorage) + מוטציות
    session.js     ניהול משתמש/הרשאות (אורח / תושב / מודרטור)

    Poi/DayClock.jsx            שעון היממה הרדיאלי ליום הגירוש
```

## תפקידים

- **אורח/ת** — צפייה בכל התכנים.
- **תושב/ת** — הוספת נקודות עניין, סיפורים ומדיה (מתפרסמים מיד).
- **מודרטור יישוב** — נעיצת יישובים, עריכת מידע כללי ומחיקת תכנים.

## מודל התוכן

כל **יישוב** מכיל מידע כללי בקטגוריות (כללי, חקלאות, חינוך, קהילה, מסחר) ורשימת **נקודות עניין**.
כל **נקודת עניין** (בית, בית כנסת, חממה) מתועדת בשלושה שלבים:

- **לפני הגירוש** — מדיה חופשית.
- **מסביב לשעון** — פריטים עם תווית שעה (מדויקת או משוערת `~`), מוצגים על שעון יממה רדיאלי.
- **אחרי הגירוש** — תחנות מתוארכות עד לבית קבע.

הזנה שיתופית בסגנון "ויקי": כמה אנשים יכולים להוסיף פריטים לאותה נקודה, ותמיד אפשר להוסיף רגע/תחנה נוספים.

## הצעדים הבאים (Backend)

הגרסה הנוכחית שומרת נתונים ב-localStorage בדפדפן (אב־טיפוס להדגמת החוויה).
לשיתופיות אמיתית צריך backend. שכבת הנתונים ב-`store.js` בנויה כ-repository מופשט כדי שיהיה קל להחליף אותה ב-API.

מומלץ ל-Hostinger: **PHP + MySQL**
- טבלאות: `settlements`, `pois`, `media`, `after_entries`, `users`.
- העלאות קבצים לתיקייה מחוץ ל-web root, עם החזרת URL.
- אימות מודרטורים + תור אישורים (`status = pending/approved`).

## פריסה

בנייה סטטית (`dist/`) שאפשר להעלות לכל אחסון.
`base: './'` ב-`vite.config.js` מאפשר הגשה גם מתת-תיקייה (למשל `example.com/gush/`).
